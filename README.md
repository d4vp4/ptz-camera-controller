# PTZ Camera TCP Controller & Telemetry Decoder

Проєкт для керування PTZ-камерою дрона (або підвісом) через локальну мережу, створений шляхом реверс-інжинірингу оригінального протоколу (на базі додатка ViewLink).

## 🚀 Можливості
- **Ініціалізація TCP-з'єднання** (порт 2000).
- **Парсинг телеметрії в реальному часі:** розшифровка кутів Yaw, Pitch та кратності оптичного зуму.
- **Детекція статусу:** розпізнавання активного руху моторчиків.
- **Асинхронне керування:** управління камерою за допомогою стрілок на клавіатурі (фоновий потік `pynput`).

## 🛠 Вимоги та встановлення
Для роботи скрипта потрібен Python 3.x та бібліотека для перехоплення натискань клавіатури:
\`\`\`bash
pip install pynput
\`\`\`

## 🧠 Деталі протоколу (Реверс-інжиніринг)
Під час розробки було розшифровано наступні особливості закритого протоколу камери:

1. **Регістр статусів (Маркери руху):**
   - `b7 ff` — камера в статичному положенні.
   - `f7 ff` — активний рух моторчиків (зум, фокус або зміна кута).

2. **Формула кутів (Yaw / Pitch):**
   Координати передаються у двох байтах (Hex -> Dec). Діапазон значень: 0-65535.
   - Якщо значення `> 32767`, воно від'ємне (віднімаємо `65536`).
   - Коефіцієнт переводу кроків моторчика в градуси: `182.0444`.

3. **Оптичний зум:**
   Передається у передостанньому байті довгого пакета телеметрії як готове значення.
   - Формула: `Зум = Значення_Байта(Dec) / 10`.
   - *Приклад: байт `64` (Hex) -> `100` (Dec) -> Зум `10.0x`.*

4. **Система керування (RC PWM):**
   Команди руху базуються на стандарті радіокерування (PWM).
   Центральне (нейтральне) положення стіка — `05 dc` (1500 мікросекунд). Відхилення від цього значення вказує напрямок та швидкість руху.
