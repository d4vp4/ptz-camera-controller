# PTZ Camera TCP Controller & Telemetry Decoder

Проєкт для керування PTZ-камерою дрона (на базі протоколу Viewpro) через локальну мережу. Створений шляхом реверс-інжинірингу оригінального закритого TCP-протоколу.

Скрипт працює як повноцінна наземна станція керування (GCS), дозволяючи керувати рухом, зумом, тепловізором та читати телеметрію в реальному часі виключно за допомогою клавіатури.

## 🚀 Можливості
- **Ініціалізація TCP-з'єднання** (порт 2000).
- **Парсинг телеметрії в реальному часі:** автоматична розшифровка кутів Yaw, Pitch та кратності оптичного зуму.
- **Детекція статусу:** розпізнавання активного руху моторчиків об'єктива або підвісу.
- **Асинхронне керування (RC-стиль):** керування камерою у фоновому потоці з автоматичним надсиланням команд зупинки при відпусканні клавіш.
- **Керування тепловізором:** перемикання між оптичним та інфрачервоним сенсорами, зміна кольорових палітр.

## ⌨️ Керування (Гарячі клавіші)

| Клавіша | Дія (Функція)                                                                                            |
| :--- |:---------------------------------------------------------------------------------------------------------|
| **`Стрілки`** (Вгору/Вниз/Вліво/Вправо) | Керування рухом камери (Pan / Tilt).                                                                     |
| **`W`** | Наблизити (Zoom In).                                                                                     |
| **`S`** | Віддалити (Zoom Out).                                                                                    |
| **`H`** | Повернення в початкове положення (Home).                                                                 |
| **`N`** | Погляд вертикально вниз (Pitch -90°).                                                                    |
| **`T`** | Увімкнути/Вимкнути тепловізор (Thermal Toggle).                                                          |
| **`P`** | Змінити палітру тепловізора (White Hot ➔ Black Hot ➔ Iron Red). Працює лише при увімкненому тепловізорі. |
| **`ESC`** | Коректне завершення роботи скрипта.                                                                      |

> **Примітка:** Команди руху та зуму автоматично зупиняються, щойно ви відпускаєте відповідну клавішу.

## 🛠 Вимоги та встановлення
Для роботи скрипта потрібен Python 3.x та бібліотека для перехоплення натискань клавіатури:
\`\`\`bash
pip install pynput
\`\`\`
Запуск скрипта:
\`\`\`bash
python camera_control.py
\`\`\`
*(Обов'язково зробіть вікно терміналу активним, щоб програма могла зчитувати натискання клавіш).*

## 🧠 Деталі протоколу (Результати реверс-інжинірингу)
Під час розробки було розшифровано наступні особливості закритого протоколу камери:

1. **Регістр статусів (Маркери руху):**
   - `b7 ff` — камера в статичному положенні.
   - `f7 ff` — активний рух моторчиків (зум, фокус або зміна кута).

2. **Формула кутів (Yaw / Pitch):**
   Координати передаються у двох байтах (Hex -> Dec). Діапазон значень: 0-65535.
   - Якщо значення `> 32767`, воно від'ємне (віднімаємо `65536`).
   - Коефіцієнт переводу кроків моторчика в градуси: `182.0444`.
   *(Наприклад: для розрахунку пакету з кутом 45°, потрібно помножити 45 на 182, що дає ~8190 або `1F FE` в Hex).*

3. **Оптичний зум:**
   Передається у передостанньому байті довгого пакета телеметрії як готове значення.
   - Формула: `Зум = Значення_Байта(Dec) / 10`. 
   - *Приклад: байт `64` (Hex) -> `100` (Dec) -> Зум `10.0x`.*

4. **Система керування (RC PWM):**
   Команди безперервного руху базуються на стандарті радіокерування (PWM). 
   Центральне (нейтральне) положення стіка — `05 dc` (1500 мікросекунд). Відхилення від цього значення вказує напрямок та швидкість руху.
